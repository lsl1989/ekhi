Redis For List

Redis - list#

Command#

Insert value into list#

rpush key value [value ...]
lpush key value [value ...]

从左(lpush -> l means left)或从右(rpush -> r means right)往列表插入数据

127.0.0.1:6379> lpush li x1 x2 x3
(integer) 6
127.0.0.1:6379> rpush li y1 y2 y3
(integer) 9
127.0.0.1:6379> lrange li 0 -1
1) "x3"
2) "x2"
3) "x1"
4) "a"
5) "c"
6) "b"
7) "y1"
8) "y2"
9) "y3"

linsert key before|after pivot value

找到list中值等于pivot的元素，并根据before|after参数来在该元素的前|后插入新值value

127.0.0.1:6379> lrange li 0 -1
1) "a"
2) "c"
3) "b"
127.0.0.1:6379> linsert li before c x
(integer) 4
127.0.0.1:6379> linsert li after c y
(integer) 5
127.0.0.1:6379> lrange li 0 -1
1) "a"
2) "x"
3) "c"
4) "y"
5) "b"

Get Value#

lrange key start stop

start和stop可以为负数，负数代表从右往左的顺序，-1是最后一个元素。 lrange命令返回值包含了stop自身，和很多应用程序的处理不太一样

127.0.0.1:6379> lrange li 0 -1
1) "a"
2) "c"
3) "b"

左右即前后，list是一个有序列表，从左往右即从前往后，反之亦然

  • 获取指定下标的元素

lindex key index

127.0.0.1:6379> LRANGE li 0 -1
1) "a"
2) "x"
3) "c"
4) "y"
5) "b"
127.0.0.1:6379> LINDEX li 2
"c"

index 从0开始

  • 获取长度

llen key

127.0.0.1:6379> llen li
(integer) 5

Delete value#

lpop key rpop key

从左从右弹出一个元素

lrem key count value

lrem 命令会从 列表 中找到 等于 value 的元素进行 删除，根据 count 的不同分为三种情况:

  • count > 0：从左到右，删除最多 count 个元素。
  • count < 0：从右到左，删除最多 count 绝对值 个元素。
  • count = 0，删除所有。

ltrim key start stop

根据start和stop的值剪切列表

修改列表中的值#

lset key index new_value

将下标为index的元素修改为新值new_value

阻塞操作#

blpop key [key ...] timeout brpop key [key ...] timeout

阻塞式的弹出list中的值，与lpop和rpop类似，从左或从右弹出。 当list不为空时，立即弹出list中的值。 当list为空时，若timeout>0则会阻塞并等待timeout的秒数后返回空值，若在等待的时间内list插入了新值，则会立即弹出新值。而若timeout=0,则会一直等待直到插入新值。

阻塞式弹出值当接受多个key的时候，会从左至右遍历key，若其中任意一个有值则会立即弹出该值

多个客户端对同一个key执行blpop，最先执行blpop的可以获取到弹出的值

List 编码#

redis3.2之前使用ziplist和linkedlist. 3.2版本后使用quicklist

ziplist#

列表元素小于list-max-ziplist-entries(默认512)，并且列表中的元素值都小于list-max-ziplist-value（默认64字节），redis会使用ziplist来减少内存使用

linkedlist#

当ziplist的条件无法满足时，redis会使用链表linkedlist来实现list

quicklist#

由于linkedlist的内存开销会比较大(每个节点上需要额外两个指针，双向链表各个节点是单独的内存块，地址不连续，节点多了容易产生内存碎片。)，而ziplist由于是连续的内存，插入和删除操作需要频繁的申请和释放内存，所以当ziplist长度很长的时候，一次realloc(C语言重新分配内存)会导致大量的数据拷贝。

quicklist的结构是在一个链表linkedlist中的每个元素中额外存放一个ziplist，在这些ziplist中存放用户数据

实际应用#

  • 利用lpush brpop的组合使用来实现消息队列
  • 用list存放列表数据
